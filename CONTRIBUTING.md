# Contributing

Thanks for poking at docforge. The goal of this guide is to get a working dev loop in under five minutes and to make the bar for a useful PR clear.

## Dev loop

```bash
python -m venv .venv
.venv/Scripts/activate            # Windows; source .venv/bin/activate on macOS/Linux
pip install -e .[dev]
pytest -q                         # full suite — should be ~10s, all green
```

Make targets cover the common commands:

```bash
make test            # full suite
make test-fast       # skips eval tests (no embedding-model warmup)
make lint            # ruff
make serve           # uvicorn on :8000 with --reload
make run REPO=/path/to/some/repo
make eval            # docforge-eval over eval/testset
```

## Where to look

- `src/docforge/scout/` — pure static analysis on the file tree.
- `src/docforge/indexer/` — tree-sitter chunking + fastembed embeddings + Chroma.
- `src/docforge/agents/` — LLM agents + the LangGraph supervisor + `_parallel.py`.
- `src/docforge/evals/` — Sonnet-judged 5-axis scoreboard harness.
- `src/docforge/server/` — FastAPI app + Jinja templates + SSE job runner.
- `eval/testset/<repo>/` — ground-truth claims for the eval.

`ARCHITECTURE.md` (repo root) is the human-written overview of how these fit together. `docs/ARCHITECTURE.md` is generated by docforge itself.

## Adding an agent

The pipeline is intentionally small. New agents earn their place by either:

1. **Adding a parallel branch from `START`** — pure static analysis, no LLM, writes a new disjoint key in `GraphState`. Good fit for: security-pattern detection, dependency-health scan, license audit, git-history reader.
2. **Sitting between two existing nodes** — needs a clear input/output contract in `agents/state.py` and a reason it's not just a tool of the existing nodes.

For every new agent:
- Add a `TypedDict` for its output in `agents/state.py`.
- Add a `run_<name>(state) -> dict` function. State updates are partial dicts; LangGraph merges by key.
- Export it from `agents/__init__.py`.
- Wire into `agents/supervisor.py`.
- Tests in `tests/test_<name>.py`. Mock LLM calls; build fixture repos in `tmp_path`.

## Adding a testset entry

Drop a new dir under `eval/testset/<repo>/` with `meta.json` (repo_path or git_url) and `claims.jsonl` (~20 hand-labelled claims). The runner picks it up automatically. See `eval/testset/README.md` for the schema.

## PR checklist

1. `make test` green.
2. `make lint` green.
3. New behavior has a test. New agent has at least one fixture-based test.
4. Anything that changes citation behavior, retrieval, or the critic loop deserves an eval comparison — run `docforge-eval` before and after, mention the delta in the PR.
5. Don't add a dependency without a one-line justification.
6. Don't add a feature flag for backwards-compatibility unless it's protecting an external interface (API endpoint, CLI flag).

## Tone

Documentation generated by this project is held to a hard rule: every non-trivial claim cites `[file:line]`. The same applies to PR descriptions. If you're making a behavior change, name the file and line you're changing.
